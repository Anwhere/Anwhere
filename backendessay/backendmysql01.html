Data truncation: Incorrect datetime value时间入库错误
MySQL & Backend

By Anwhere Jin. Date issued : 11-16-2018.

在开发一个网上商城时，实现保存商品的方法，无法往数据库商品表插入数据，报错信息显示插入的数据格式不对，查看日志显示为‘pdate’的格式不对，但是我们将日志文件打印出来，发现日期格式看起来并没有任何错误，简单查询了一下，发现是Mysql数据库的问题。


· 01>>> Eclipse报错信息

Data truncation: Incorrect datetime value: ‘’ for column ‘pdate’ at row 1

在开发一个网上商城时，实现保存商品的方法，无法往数据库商品表插入数据，报错信息显示插入的数据格式不对-- ‘pdate’


· 02>>> 错误原因

a. java中Date类型默认是格林尼治时间，对应数据表使用的是datetime类型

b. 我们引入的mysql的jar包，数据库驱动mysql-connector-java封装了将java的Date类型的格林尼治时间转换为‘yyyy-MM-dd HH:mm:ss’的功能。对应的数据表使用的是date类型

c. 当我们的MySQL版本和驱动mysql-connector-java的jar包版本一致才能够转换成功（我的mysql版本为5.6，驱动是5.1.7）


· 03>>> 解决方案

a. 更改数据库版本与jar包一致

b. 对pdate使用SimpleDateFormat进行转换，转换成和数据库一致的格式：yyyy-MM-dd hh:mm:ss

· 04>>> 我们的解决方式

我们采用解决方案2，现展示修改前后的代码

修改前：

  // 将提交的数据添加到数据库中。
  product.setPdate(new Date());

修改后：

  // 将提交的数据添加到数据库中。
  SimpleDateFormat sf = new SimpleDateFormat("yyyy-MM-dd hh:mm:ss");
  String format = sf.format(new Date());
  Date parse = sf.parse(format);
  System.out.println("1:"+new Date());
  System.out.println("2:"+parse);
  product.setPdate(parse);

问题顺利解决，查看日志，将修改前后的日期都打印出来，可以看到两者并无视觉上的差别，实际上内部的数据格式已经发生变化。
